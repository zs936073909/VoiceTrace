## 一、安装

### 系统要求

- **操作系统**：Windows 10 / 11（64 位）
- **硬盘空间**：约 500 MB（安装后）
- **内存**：建议 4 GB 以上
- **麦克风**：任意 USB 麦克风或笔记本内置麦克风

### 安装步骤

1. 双击运行 `VoiceTrace_Setup_v1.0.0.exe`
2. 安装向导启动后，点击「下一步」
3. 选择安装目录（默认 `C:\Program Files\VoiceTrace`，也可装到 D 盘）
4. 勾选是否创建桌面快捷方式和开始菜单快捷方式
5. 点击「安装」→ 等待几秒 → 点击「完成」
6. 可选：勾选「立即启动 VoiceTrace」直接打开程序

### 卸载

- 从「开始菜单 → VoiceTrace → 卸载 VoiceTrace」卸载，或
- 从「设置 → 应用 → VoiceTrace」卸载
- 卸载不会删除你的训练数据（保存在 `C:\Users\你的用户名\.voicetrace\`）

## 二、快速上手（5 分钟开始训练）

### 第 1 步：新建稿件

打开软件 → 切换到「**稿件管理**」标签 → 点击「**新建稿件**」：

- **标题**：例如「新闻播报练习 01」
- **分类**：新闻播报 / 即兴评述 / 模拟主持 / 自定义
- **语言**：中文 / 英文 / 中英混合
- **内容**：粘贴你要练习的稿件全文

点击「保存」。

### 第 2 步：录音

切换到「**录音**」标签 → 程序会自动加载你刚选的稿件。

- 点击「**开始录音**」或按 `Ctrl+R`
- 对着麦克风朗读稿件
- 朗读中遇到卡壳、读错的地方，按「**空格**」标记卡顿
- 朗读完毕按「**停止**」或 `Ctrl+R`

录音会自动保存，并自动记一次训练打卡。

### 第 3 步：分析

切换到「**分析**」标签 → 选择刚录的音 → 点击「**分析**」。

几秒后你会看到：

- **语速**（CPM = 字/分钟，WPM = 词/分钟）
- **卡顿次数**与总停顿时长
- **能量**（音量稳定性参考）
- **是否符合标准**（绿勾 = 符合，红叉 = 偏快/偏慢）
- **语速趋势图**（多次录音的对比）
- **波形图**（红色竖线 = 你标记的卡顿点）
- **逐句分析表**（每句的语速、停顿）

### 第 4 步：跟读练习（可选）

切换到「**跟读模式**」标签：

1. 从下拉框选一条「示范录音」（可以是老师录的，也可以是你之前录得最好的）
2. 点击「**播放示范**」听一遍
3. 点击「**开始跟读录制**」跟着读
4. 停止后系统会自动对比，给出**相似度百分比**和改进建议

### 第 5 步：查看进度

切换到「**训练打卡**」标签：

- 总训练次数、总时长、连续打卡天数
- 日历视图（红色 = 已打卡）
- 训练记录列表

## 三、功能详解

### 1. 稿件管理

| 功能 | 说明 |
|---|---|
| 新建/编辑/删除稿件 | 支持中文、英文、中英混合 |
| 4 种分类 | 新闻播报、即兴评述、模拟主持、自定义 |
| 稿件预览 | 选中后下方显示内容前 500 字 |
| 自定义语速标准 | 点击「自定义标准」按钮，添加你自己的语速区间 |

**内置语速标准**（基于播音主持行业经验值）：

| 分类 | 中文 (CPM) | 英文 (WPM) |
|---|---|---|
| 新闻播报 | 250–300 | 170–190 |
| 即兴评述 | 200–280 | 140–180 |
| 模拟主持 | 220–300 | 150–190 |

> 浙江传媒大学双播专业的朋友可以根据老师要求自定义标准。

### 2. 录音

| 功能 | 说明 |
|---|---|
| 录音格式 | WAV 16kHz 单声道（适合语音分析） |
| 最长时长 | 5 分钟（自动停止） |
| 卡顿标记 | 朗读时空格键即时标记，支持多个 |
| 降噪选项 | 勾选「分析时启用降噪」可减少环境噪声影响 |
| 录音回放 | 录完即可播放，带进度条拖动 |
| 卡顿跳转 | 回放时点击卡顿时间按钮直接跳到该位置 |

### 3. 分析

分析标签页包含 3 个子标签：

#### 语速趋势
- 折线图展示同一录音的多次分析结果
- 横轴 = 第几次分析，纵轴 = 语速
- 用于跟踪你针对同一稿件的进步轨迹

#### 波形图
- 红色波形 = 你的声音能量
- 红色竖线 = 你标记的卡顿点
- 可直观看到卡顿分布

#### 逐句分析
- 按稿件标点自动切分句子
- 每句显示：字数、语速、停顿次数、停顿时长
- 用于精确定位哪一句读得太快/太慢/卡壳

### 4. 跟读模式

跟读模式是**核心训练功能**：

1. **选示范**：从录音列表选一条作为「标杆」
2. **听示范**：播放示范录音，带进度条
3. **录跟读**：录下你的跟读
4. **自动对比**：系统用 MFCC 音色特征 + 语速 + 卡顿三个维度对比
5. **看报告**：
   - 相似度百分比（0–100%）
   - 你的语速 vs 示范语速
   - 优点（绿色）/ 待改进（红色）

> **使用建议**：让老师或学长录一条「示范录音」存进系统，之后你随时可以跟读对比，相当于 24 小时陪练。

### 5. 对比

对比标签用于比较**任意两条录音**：

- 选「当前录音」和「基准录音」
- 点击「对比」生成报告
- 报告会保存到数据库，可日后导出

适合场景：
- 比较你不同时期的录音，看进步
- 比较你和同学的同一稿件
- 比较你跟老师示范的差距

### 6. 训练打卡

| 组件 | 说明 |
|---|---|
| 统计卡片 | 总训练次数、总时长、连续打卡天数、今日次数 |
| 打卡日历 | 红色日期 = 有训练记录，可翻看上下月 |
| 手动打卡 | 没录音但想记一次训练时用（例如线下练习） |
| 训练记录表 | 最近 50 条训练记录，含稿件、时长、备注 |

> 每次录音会**自动打卡**，无需手动操作。

### 7. 数据导出

支持 3 种格式：

| 格式 | 用途 |
|---|---|
| **CSV** | Excel 打开，适合做数据分析、画图表 |
| **JSON** | 程序员友好，适合二次处理 |
| **PDF** | 正式报告，适合交给老师/家长查看 |

可导出内容：
- 所有录音分析（含语速、卡顿、能量等指标）
- 所有稿件（标题、分类、语言、字数）
- 所有对比记录（相似度、差异）

### 8. 主题切换

- 菜单栏「**视图 → 切换深色主题**」
- 或按 `Ctrl+D`
- 浅色：Editorial 风格，米白底 + 红色强调
- 深色：深蓝底 + 玫红强调，适合夜间练习

## 四、台风训练

台风训练是 v2.0 新增的核心功能，通过摄像头实时分析你的**镜头感**和**肢体语言**，帮助你形成专业台风。

### 前置要求

- 摄像头（笔记本内置或外接 USB 摄像头均可）
- MediaPipe 库（首次使用时按提示安装，或手动运行 `pip install mediapipe opencv-python`）

### 训练维度

台风训练从 **6 个维度** 评估你的表现：

| 维度 | 说明 | 评分标准 |
|---|---|---|
| **眼神交流** | 是否直视镜头 | 虹膜偏移量 < 10% 为佳 |
| **表情管理** | 微笑度与紧张度 | 自然微笑 + 放松面部为佳 |
| **头部姿态** | 头部偏转/点头/歪头 | 平均偏转 < 10° 为佳 |
| **站姿/坐姿** | 肩膀倾斜、身体前倾 | 倾斜 < 5° 为佳 |
| **手势** | 手势时长占比 | 站姿 20%-60%，坐姿 10%-50% 为佳 |
| **稳定性** | 身体晃动幅度 | 平均运动量 < 0.01 为佳 |

### 坐姿 vs 站姿模式

软件支持两种训练模式，针对不同场景优化：

| 模式 | 适用场景 | 特殊检测 |
|---|---|---|
| **坐姿** | 电脑前日常训练 | 探颈检测（"龟颈"问题） |
| **站姿** | 模拟演讲/汇报 | 身体倾斜、手势幅度 |

> 日常用电脑练习选「坐姿」，正式模拟演讲选「站姿」。

### 使用步骤

1. 切换到「**台风训练**」标签
2. 选择训练模式（坐姿/站姿）和摄像头
3. 勾选「镜像显示」可更自然地调整姿势
4. 可选：设置训练时长（0 = 手动停止）
5. 点击「**开始训练**」
6. 对着摄像头进行朗读/演讲练习
7. 实时查看右侧反馈：
   - **实时反馈**：眼神、表情、头部的即时评价
   - **摄像头叠加层**：画面上显示各项指标
8. 点击「**停止训练**」结束
9. 查看综合评分（雷达图）和详细数据
10. 点击「**保存结果**」记录到历史

### 实时反馈说明

训练时画面会叠加显示：

- **Eye Contact**：眼神交流百分比（绿色 = 良好，红色 = 偏离）
- **Smile**：微笑度
- **Head Yaw**：头部左右偏转角度
- **Gesturing**：是否在做手势
- **Posture**：姿态是否良好
- **警告文字**：如「请看向镜头」「表情紧张，请放松」

### 雷达图解读

训练结束后，右侧雷达图展示 6 维评分（0-100）：

- 图形越饱满，台风越均衡
- 某一维度凹陷，说明该维度需要加强
- 综合评分 = 6 维平均分

### 训练历史

底部表格记录所有台风训练历史，包含日期、时长、各维度评分和综合评分。

## 五、文案写作

文案写作是 v2.0 新增功能，帮助你快速生成播音练习稿件。支持两种生成方式：

### 方式一：模板生成（离线可用）

内置 9 套专业模板，覆盖常见播音场景：

| 类别 | 模板 |
|---|---|
| 新闻播报 | 时政新闻、民生新闻 |
| 即兴评述 | 观点类、现象类 |
| 模拟主持 | 晚会主持、访谈主持 |
| 演讲 | 励志类 |
| 英文 | News Broadcast、Impromptu Commentary |

**使用步骤**：

1. 切换到「**文案写作**」标签 → 「模板生成」页
2. 选择类别（新闻播报/即兴评述/模拟主持/演讲）
3. 选择语言（中文/英文）
4. 选择具体模板
5. 在「填写内容」区填入占位符（如日期、主题、细节等）
6. 点击「**生成文案**」
7. 生成的文案会显示在底部，附带写作提示（语速建议、注意事项）
8. 可「复制到剪贴板」或「保存为稿件」

> 每个模板都附带**写作提示**，告诉你该类稿件的语速范围和表达要点。

### 方式二：AI 智能生成（需联网）

支持任何兼容 OpenAI 格式的 API（OpenAI、DeepSeek、Moonshot、通义千问等）。

**配置步骤**：

1. 切换到「**AI 生成**」页
2. 填写 API 配置：
   - **API 地址**：如 `https://api.openai.com/v1/chat/completions`
   - **API 密钥**：你的 API Key
   - **模型**：如 `gpt-3.5-turbo`、`deepseek-chat` 等
   - **创造性**：0-1，越高越有创意，0.7 为推荐值
3. 填写生成参数：
   - 文案类型、语言、主题、时长、风格
   - 附加要求（可选）
4. 点击「**AI 生成文案**」

**推荐 API 服务**：

| 服务 | 特点 | 价格 |
|---|---|---|
| DeepSeek | 国产，中文效果好，便宜 | 约 ¥1/百万 token |
| Moonshot (Kimi) | 长文本能力强 | 约 ¥12/百万 token |
| 通义千问 | 阿里云，稳定 | 约 ¥4/百万 token |
| OpenAI | 效果最佳 | 约 $2/百万 token |

> **隐私说明**：API 密钥仅保存在本地内存，不会上传或存储到任何服务器。

### 保存为稿件

无论哪种方式生成的文案，都可以点击「**保存为稿件**」直接导入稿件管理，方便后续录音练习。

## 七、常见问题（FAQ）

### Q1：录音没有声音 / 波形是平的？

- 检查 Windows 麦克风权限：设置 → 隐私 → 麦克风 → 允许桌面应用访问
- 检查麦克风是否被其他软件占用（如微信、腾讯会议）
- 换一个 USB 麦克风试试

### Q2：语速显示为 0 或异常？

- 确保稿件内容**与实际朗读内容一致**（语速 = 字数 ÷ 朗读时长）
- 稿件语言选对（中文稿件选「中文」，不要选「英文」）
- 录音时长至少 10 秒以上，太短分析不准

### Q3：分析时报错？

- 确保录音文件存在（不要手动删除 `.voicetrace/recordings/` 里的文件）
- 关闭其他占用 CPU 的程序后重试
- 如果是降噪报错，取消勾选「降噪」再试

### Q4：跟读相似度总是很低？

- 相似度基于 MFCC 音色特征，受以下因素影响：
  - 麦克风型号不同（你和示范用同一个麦最好）
  - 录音环境不同
  - 性别/音色差异（这是正常的，不必追求 100%）
- **重点看语速差和卡顿差**，比相似度更有参考价值

### Q5：数据存在哪里？重装会丢吗？

- 数据目录：`C:\Users\你的用户名\.voicetrace\`
  - `broadcast.db`：数据库（稿件、录音、分析、打卡）
  - `recordings/`：所有 WAV 录音文件
- **卸载软件不会删除数据**
- 重装系统前请备份这个目录

### Q6：可以处理 MP3 吗？

- 当前版本只支持 WAV 录音（保证分析质量）
- 如需导入 MP3，可先用格式工厂等工具转成 WAV

### Q7：安装时被 Windows Defender 拦截？

- 这是未签名软件的常见提示，本软件**完全安全开源**
- 点击「更多信息」→「仍要运行」即可
- 如需数字签名版本，需购买代码签名证书（收费）

### Q8：能在 Mac 上用吗？

- 当前安装包仅支持 Windows
- Mac 用户可从源码运行（需 Python 3.10+ 和依赖库）

## 八、技术说明

### 技术栈

| 组件 | 技术 |
|---|---|
| 界面框架 | PySide6 (Qt 6) |
| 音频分析 | librosa、webrtcvad、pydub |
| 台风分析 | MediaPipe Face Mesh + Pose |
| 机器学习 | scikit-learn（余弦相似度） |
| 数据库 | SQLite 3 |
| 报告导出 | reportlab（PDF）、原生 CSV/JSON |
| 打包 | PyInstaller + Inno Setup |

### 核心算法

- **语速计算**：`字数 ÷ 实际说话时长 × 60`
- **实际说话时长**：`总时长 − VAD 检测到的停顿时长`
- **停顿检测**：WebRTC VAD（ aggressiveness = 2），30ms 帧级判断
- **音色相似度**：13 维 MFCC 均值向量的余弦相似度
- **降噪**：谱减法（取前 0.5 秒作为噪声样本）
- **逐句分析**：按标点切分 + 时长均分（简化方案，非强制对齐）
- **眼神交流**：MediaPipe Face Mesh 虹膜追踪，计算虹膜中心相对眼眶的偏移量
- **表情分析**：嘴角宽度/眼嘴距离比值计算微笑度；眉毛-眼睛距离计算紧张度
- **头部姿态**：基于 2D 关键点的 yaw/pitch/roll 近似估计
- **身体姿态**：MediaPipe Pose 33 点，肩膀倾斜 + 探颈检测（坐姿）
- **手势识别**：手腕高度相对髋部-肩部的归一化位置

### 数据目录结构

```
~/.voicetrace/
├── broadcast.db          # SQLite 数据库
└── recordings/           # WAV 录音文件
    ├── 新闻播报_20260617_143022.wav
    └── follow_20260617_150030.wav
```

### 数据库表

| 表名 | 用途 |
|---|---|
| scripts | 稿件 |
| recordings | 录音 |
| stumbles | 卡顿标记 |
| analyses | 分析结果（含 MFCC、谱特征、逐句 JSON） |
| comparisons | 对比记录 |
| training_sessions | 训练打卡 |
| custom_standards | 自定义语速标准 |
| posture_records | 台风训练记录（眼神、表情、姿态等评分） |
| script_templates | 文案模板 |
